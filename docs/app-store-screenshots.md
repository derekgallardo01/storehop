# App Store screenshots — what's in `/tmp/storehop-app-store/`

Generated by running `DesignSystemTourTest/testLightAppearanceTour` against two simulators and extracting the PNG attachments from each xcresult bundle. Two device sizes, six screens each, matching the App Store Connect upload form's required + recommended sizes.

| Index | Story beat | Why it's here |
|---|---|---|
| 01 | StorePicker — Lidl + Aldi + "+ Add store" FAB + Settings/Edit toolbar | First-launch hero shot: clean, branded, shows the multi-store premise immediately |
| 02 | Shop at Lidl — "Eggs" (Lidl-only) + "Milk" (Dairy) with sort/check/more toolbar | Demonstrates the per-store list, the toolbar's three controls, and the cross-category grouping |
| 03 | Items list — Bread / Eggs / Milk with the +/- toggles | Master library view; `+/-` shows the v0.6.1 toggle including the disabled state on Bread (no stores yet) |
| 04 | Bulk-tag selection mode — "1 selected" + storefront action | v0.8.1 hero: long-press → contextual toolbar with the new "Tag to stores…" action |
| 05 | Bulk-tag sheet — "Tag selected items" + Cancel/Add stores | Completes the v0.8.1 bulk-tag story; demonstrates multi-store assignment |
| 06 | Edit item form — Milk with category picker, stores chips, toggles | Form depth: category picker, "+ New category" inline, store chips, "Always on the list" / "Critical" toggles |

## How to regenerate

```sh
cd ios && xcodegen generate
for device in 'iPhone 17 Pro Max' 'iPhone 17'; do
  bundle="/tmp/tour-${device// /-}.xcresult"
  rm -rf "$bundle"
  xcodebuild test -project Storehop.xcodeproj -scheme Storehop \
    -destination "platform=iOS Simulator,name=$device" \
    -only-testing:StorehopUITests/DesignSystemTourTest/testLightAppearanceTour \
    -resultBundlePath "$bundle" \
    CODE_SIGNING_ALLOWED=NO
done
```

The xcresult `Data/` dirs hold the PNGs (un-extension'd, content-hashed names). Use `file(1) | grep PNG` to find them.
